# 🚩 里程碑规划模板：从切片到金盘

**文档目标**：定义 Project Vampirefall 的宏观开发时间轴。设定每个阶段的**验收标准 (Exit Criteria)**，防止项目无休止延期。

---

## Phase 1: 垂直切片 (Vertical Slice)
**目标**：验证核心玩法是否好玩。做“一厘米宽，一公里深”的内容。
**周期**：2-3 个月。

### 包含内容
*   **1 个角色**：全套动作、技能、特效。
*   **1 种敌人**：基础 AI、受击反馈。
*   **1 个关卡**：完整的刷怪循环、地形阻挡。
*   **核心循环**：战斗 -> 升级 -> 选技能 -> 变强 -> 战斗。

### 验收标准 (Exit Criteria)
1.  [ ] 核心战斗手感得到团队内部一致认可（打击感、流畅度）。
2.  [ ] 数值闭环：能跑通 10 分钟的流程，没有明显卡点。
3.  [ ] 性能达标：在目标机型上稳定 60 FPS。
4.  [ ] **必须包含 UI/UX**：哪怕是简陋的，也要有完整的 HUD 和结算界面。

---

## Phase 2: Alpha (量产阶段)
**目标**：铺量。填充内容，完成所有核心系统。
**周期**：4-6 个月。

### 包含内容
*   **所有角色与技能**：不仅是配置，而是实装。
*   **所有关卡与怪物**：完成美术资产置换 (Greyboxing -> Final Art)。
*   **Meta 系统**：局外天赋树、商店、成就系统。
*   **新手引导**：第一版教程关卡。

### 验收标准
1.  [ ] Feature Complete (功能完备)：所有策划案里的功能都写进了代码里。
2.  [ ] 主流程通关：可以从头玩到尾，不会 Crash。
3.  [ ] 占位符清零：游戏中不再出现白色方块或 "TODO" 文本。

---

## Phase 3: Beta (打磨与测试)
**目标**：修 Bug，调数值，优化体验。**严禁添加新功能**。
**周期**：2-3 个月。

### 关键动作
*   **Playtest (试玩测试)**：邀请外部玩家（亲友团、Discord 社区）进行测试。
*   **Bug Bash (扫除)**：全员停下开发，集中修 Bug。
*   **数值调优**：根据后台埋点数据，调整怪物的血量、掉率。
*   **本地化 (L10N)**：导出文本进行翻译。

### 验收标准
1.  [ ] Zero Critical Bugs (零崩溃 Bug)。
2.  [ ] 性能优化完成：低端机型适配。
3.  [ ] 或者是 Steam Playtest 上线。

---

## Phase 4: Gold (上市)
**目标**：发布 v1.0 版本，准备 Day 1 Patch。

### 关键动作
*   **Store Page**：Steam/AppStore 页面素材准备（宣传片、截图）。
*   **Review Copy**：发给媒体和主播的评测码。
*   **Live Ops 准备**：配置好云端开关，准备好后续活动的资源。

---

## 风险评估清单 (Risk Assessment)

在每个里程碑开始前，必须盘点风险：

| 风险项 | 可能性 | 影响程度 | 应对方案 (Plan B) |
| :--- | :--- | :--- | :--- |
| 美术外包延期 | 高 | 高 | 使用 Asset Store 资源先顶替，保证逻辑开发不中断。 |
| 联机同步写不出来 | 中 | 极高 | 砍掉联机，转纯单机（最坏情况）。 |
| 审核被拒 (iOS/Console) | 中 | 高 | 预留 2 周缓冲期，提前提交审核。 |
