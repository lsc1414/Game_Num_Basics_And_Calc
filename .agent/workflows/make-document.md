---

1.  **确认研究主题**
    *   明确用户需要研究的游戏机制或系统（例如：仇恨系统、掉落机制、寻路算法）。
    *   确定文档保存路径（通常在 `Design/` 或 `Tech/` 目录下）。

2.  **生成文档结构**
    *   创建一个新的 Markdown 文件，包含以下标准章节：

    ```markdown
    # 🧙‍♂️ [主题名称] 深度研究

    ## 📚 1. 理论基础 (Theoretical Basis)
    *   **核心定义**: 该机制的学术或通用定义。
    *   **数学模型**: 相关的公式、曲线或算法（如适用）。
    *   **设计心理学**: 该机制如何影响玩家体验（心流、奖励反馈等）。

    ## 🛠️ 2. 实践应用 (Practical Implementation)
    *   **Vampirefall 适配**: 结合本项目（塔防+肉鸽+Looter）的特性进行设计。
    *   **数据结构**: 建议的 C# 类结构或 JSON 配置格式。
    *   **伪代码/逻辑**: 关键算法的实现思路。
    *   **Unity 实现**: 具体的组件设计或性能注意事项。

    ## 🌟 3. 业界优秀案例 (Industry Best Practices)
    *   **案例分析**: 选取 2-3 个知名游戏（如《Hades》、《Slay the Spire》、《Kingdom Rush》等）的同类机制。
    *   **优缺点对比**: 分析它们为什么好，以及潜在的缺陷。
    *   **借鉴点**: 我们应该模仿什么，规避什么。

    ## 🔗 4. 参考资料 (References)
    *   📄 相关论文 (Papers)
    *   📺 GDC 演讲 / 视频分析
    *   🌐 技术博客 / Wiki 链接
    ```

3.  **撰写内容要求**
    *   **语言**: 必须使用 **中文**。
    *   **图标**: 标题和关键点必须使用 Emoji 图标以增强可读性。
    *   **兼容性**: 确保所有数学公式和 Mermaid 图表符合 `GEMINI.md` 中的 GitHub Pages 兼容性要求。
    *   **详细举例**: 每个部分都必须包含具体的例子，不能只有空泛的理论。

4.  **自我审查**
    *   检查是否涵盖了理论、实践和案例三个维度。
    *   检查是否包含具体的参考链接。

5.  **更新索引**
    *   打开 `readme.md` 文件。
    *   找到最相关的分类（例如 "1. 游戏设计 (Design)" 下的 "⚔️ 核心机制" 或 "🎓 理论与策略"）。
    *   添加新文档的链接，格式为：`*   **[文档标题](相对路径):** 简短描述。`
    *   保持列表的整洁和对齐。
